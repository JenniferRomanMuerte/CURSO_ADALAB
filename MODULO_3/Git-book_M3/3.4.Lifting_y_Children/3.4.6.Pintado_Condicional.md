# 3.4.6 Pintado condicional

### Pintado condicional

En esta micro lección aprenderemos algo que ya hemos dejado caer en algún momento pero que queremos explicar con más detalle.

> **Nota:** es más importante descansar que leer esta lección.

### Pintado condicional en JSX

Supongamos que tenemos un componente que tiene que pintar un mensaje para informar a la usuaria de si estamos en modo edición o no. El código del componente podría ser:

```jsx
// Fichero src/components/EditMode.js

const EditMode = (props) => {
  return <p>Editing mode is {props.isEditMode ? 'ON' : 'OFF'}...</p>;
};

export default EditMode;
```

Otra opción igual de válida es:

```jsx
// Fichero src/components/EditMode.js

const EditMode = (props) => {
  return props.isEditMode ? (
    <p>Editing mode is ON ...</p>
  ) : (
    <p>Editing mode is OFF ...</p>
  );
};

export default EditMode;
```

Os queremos explicar esto porque es una forma de trabajar que veréis mucho en código ajeno y porque queremos que lo uséis en vuestros proyectos.

Por cierto esto nos viene muy bien cuando tenemos que pintar un listado de elementos y el array está vacío. Estaría guay pintar aquí el mensaje "No hay elementos para mostrar", o lo que te apetezca.
